# Memory-Bounded Search Algorithms – Code Repository

This repository contains implementations of: **A***, **IDA***, and **SMA*** for the **8-puzzle problem**, along with experiment scripts used for **benchmarking** performance.

## Quick Start

```bash
git clone https://github.com/subhikshamkrish/fai_capstone_project.git
cd fai_capstone_project/search
pip install matplotlib
python experiments/nodes_expanded_experiment.py
python experiments/memory_experiment.py`
```

## How to Run

#### Requirements
Python 3.8 or higher

Clone the repository:
`git clone https://github.com/subhikshamkrish/fai_capstone_project.git`
`cd fai_capstone_project/search`

Install dependencies: `pip install matplotlib`

---
From the root directory, run the two experiments. Plots will open in a pop up window and will also be saved to the `plots/` folder.


#### Experiment 1: Memory Tradeoff (SMA*)

Evaluates SMA* under varying memory limits and compares it to A*: 

command: `python experiments/memory_experiment.py`

**Config parameters (inside the script):**
- DEPTH: Fixed solution depth (default: 20)
- INSTANCES_PER_DEPTH: Number of instances per depth (default: 3)
- memory_fractions: Range from 0.20 to 1.50 with step 0.05 (sma memory_limit = fraction × A* peak memory) 
- Heuristic: Perturbed Manhattan heuristic (configured at import)

Output:
- Console logs (memory sma/A* ratio, nodes expanded sma/A* ratio, pruned nodes)
- Plot `sma_memory_ratio.png`  memory sma/A* ratio vs nodes expanded sma/A* ratio


#### Experiment 2: Nodes Expanded Benchmark (A*, IDA*, SMA*)

Runs all three algorithms on 8-puzzle instances of increasing solution depth and compares performance.

command: `python experiments/nodes_expanded_experiment.py`

Config parameters (inside the script):
- DEPTHS: Range of solution depths tested (default: 2 to 20)
- INSTANCES_PER_DEPTH: Number of instances per depth (default: 3)
- SMA_MEMORY_FACTOR: sma memory limit = SMA_MEMORY_FACTOR * depth
- Heuristic: Perturbed Manhattan heuristic (configured at import)

Output:
- Console logs (prints metrics for each algo: nodes expanded, runtime, peak nodes stored/nodes pruned)
- Plot `a_ida_sma_nodes.png`  Nodes Expanded vs Solution Length

## Code Structure

The repository is organized into three main components: 

1. search algorithms
2. problem setup 
3. experiments

**Extensibility:**
- The codebase separates algorithm implementations, problem definitions and experimental evaluation, promoting modularity and reuse.
- The use of an **abstract `SearchProblem` interface** decouples algorithms from the underlying domain, allowing them to be easily extended to new problems or heuristics.
- The modular experiment scripts and **configurable parameters** support systematic experimentation and reproducible benchmarking.
---

### `search/`
Contains implementations of all search algorithms:
- `a_star.py`: Standard A* search using a priority queue (`f = g + h`) with duplicate detection and cost tracking.
- `ida.py`: Iterative Deepening A*, implemented using recursive depth-first search with increasing cost threshold.
- `sma.py`: Simplified Memory-Bounded A*, includes node pruning when memory limit reached, backed-up f-values and re-expansion of forgotten nodes.

---

### `setup/`
Defines the problem domain and shared utilities:
- `benchmark_problem.py`: Abstract `SearchProblem` interface used by all algorithms.
- `eight_puzzle_benchmark.py`: Implementation of the 8-puzzle problem, including state representation, successor function and heuristics (Manhattan and perturbed Manhattan).
- `instance_generator.py`: Generates puzzle instances with exact solution depths for controlled experiments.
- `metric.py`: Tracks performance metrics such as nodes expanded, runtime, memory usage and pruning statistics.

---

### `experiments/`
Contains scripts for running evaluations:
- `nodes_expanded_experiment.py`: Compares A*, IDA* and SMA* across increasing solution depths, measuring nodes expanded and runtime.
- `memory_experiment.py`: Evaluates SMA* under varying memory limits and compares performance against A*.

---

### `plots/`
Stores output graphs generated by experiment scripts (created at runtime).
